# 🎬 CineMatch — AI-Powered Movie Recommendation System

> Discover movies you'll love — search, explore, and get personalised, AI-driven recommendations.

CineMatch is a full-stack movie recommendation web app that blends a **content-based ML model** (TF-IDF + cosine similarity) with **live data from TMDB** to deliver poster-rich recommendations, genre discovery, and a Netflix-style browsing experience — all wrapped in a custom dark "cinema" UI.

---

## ✨ Features

- **🏠 Home Feed** — Trending, Popular, Top Rated, Now Playing, and Upcoming movies, pulled live from TMDB
- **🔍 Smart Search** — Keyword search with instant suggestion dropdown and a full results grid
- **🤖 AI Picks** — Content-based recommendations using **TF-IDF cosine similarity** over a local dataset of ~45,000 movies
- **🎭 Genre Discovery** — "More like this" recommendations sourced from TMDB's `discover` endpoint based on a movie's primary genre
- **📄 Movie Details Page** — Poster, backdrop, overview, genre tags, and release year, with safe fallbacks if data is missing
- **🎨 Custom UI** — Dark, cinema-themed Streamlit interface with a responsive poster grid, hover effects, and gold accents

## 🛠️ Tech Stack

| Layer | Technology |
|---|---|
| **Backend / API** | FastAPI, Uvicorn, httpx (async TMDB calls), Pydantic |
| **ML / Recommendation Engine** | scikit-learn (`TfidfVectorizer`), pandas, NumPy, SciPy (sparse cosine similarity) |
| **Frontend** | Streamlit |
| **External Data** | [TMDB API](https://www.themoviedb.org/documentation/api) (live metadata & posters) |
| **Training Data** | [The Movies Dataset](https://www.kaggle.com/datasets/rounakbanik/the-movies-dataset) (Kaggle) — ~45K movies metadata |
| **Deployment** | Render (backend) |

## 🧠 How the Recommendations Work

1. A TF-IDF vectorizer is fit offline on movie metadata (title/overview/genres) and serialized to `tfidf.pkl` / `tfidf_matrix.pkl`.
2. A `title → row index` lookup (`indices.pkl`) maps any movie title to its row in the precomputed TF-IDF matrix.
3. At request time, the API computes cosine similarity between the queried movie's TF-IDF vector and every other row, then returns the top-N most similar titles.
4. Each recommended title is cross-referenced against TMDB to attach a poster, rating, and release date — so even a pure offline ML model ends up with a rich, visual UI.

## 🔌 API Endpoints

| Method | Endpoint | Description |
|---|---|---|
| `GET` | `/health` | Health check |
| `GET` | `/home` | Home feed by category (`trending`, `popular`, `top_rated`, `now_playing`, `upcoming`) |
| `GET` | `/tmdb/search` | Raw TMDB keyword search (multiple results, for suggestions + grid) |
| `GET` | `/movie/id/{tmdb_id}` | Full movie details by TMDB ID |
| `GET` | `/recommend/genre` | Genre-based recommendations for a given TMDB movie ID |
| `GET` | `/recommend/tfidf` | Raw TF-IDF recommendations for a given title (debug/utility route) |
| `GET` | `/movie/search` | **Bundle endpoint** — movie details + TF-IDF recs + genre recs in one call |

Interactive API docs are auto-generated by FastAPI at `/docs` (Swagger UI) once the server is running.

## 🚀 Getting Started

### Prerequisites
- Python 3.10+
- A free [TMDB API key](https://www.themoviedb.org/settings/api)
- [`uv`](https://github.com/astral-sh/uv) for environment management (or plain `venv`/`pip`)

### 1. Clone the repo
```bash
git clone https://github.com/<your-username>/movie_recommend_system.git
cd movie_recommend_system
```

### 2. Set up the environment
```bash
uv venv
.venv\Scripts\activate        # Windows
uv pip install -r requirements.txt
```

### 3. Configure environment variables
Create a `.env` file in the project root:
```
TMDB_API_KEY=your_tmdb_api_key_here
```

### 4. Run the backend
```bash
uvicorn main:app --reload
```
API docs available at `http://127.0.0.1:8000/docs`.

### 5. Run the frontend
Update `API_BASE` in `app.py` to `http://127.0.0.1:8000` for local development, then:
```bash
streamlit run app.py
```

## 📦 Note on Large Files

`movies_metadata.csv`, `df.pkl`, `tfidf_matrix.pkl`, and `tfidf.pkl` are large binary/data files. For a cleaner repo, consider either:
- Tracking them with [Git LFS](https://git-lfs.com/), or
- Excluding them from version control and providing a small preprocessing script/notebook to regenerate them from the raw dataset.
